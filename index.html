Эффективность кэша, полные сканирования таблиц, временные файлы на диске, расхождения в оценках планировщика.

Мёртвые строки, раздутие таблиц и индексов, актуальность статистики, нагрузка на автовакуум.

Безопасность

Почему это безопасно

Диагностический скрипт — открытый код. Любой специалист вашей службы безопасности может проверить его за 15 минут.

👁

Только метаданные

Скрипт читает системные каталоги PostgreSQL (pg_stat_*, pg_class, pg_index). Содержимое таблиц не затрагивается.

📖

Режим только чтение

Соединение открывается с флагом readonly и autocommit. Ни одна строка данных не может быть изменена.

🔓

Открытый код на GitHub

Весь код скрипта опубликован на GitHub под лицензией MIT. Каждый SQL-запрос виден, никаких скрытых действий.

🚫

Никакого удалённого доступа

Мы не подключаемся к вашим серверам. Вы запускаете скрипт сами и передаёте только файл с результатом через форму на сайте.

Что делает диагностический скрипт (фрагмент)
# Соединение — только чтение conn.set_session(readonly=True, autocommit=True) cur.execute("SET search_path = pg_catalog, public;") cur.execute("SET statement_timeout = %s;", (15000,))
# Пример собираемой информации → pg_stat_user_tables # размер, сканирования → pg_stat_user_indexes # использование индеков → pg_settings # конфигурация → pg_class # структура объектов
В скрипте нет SELECT FROM ваших таблиц — только системные представления PostgreSQL.
